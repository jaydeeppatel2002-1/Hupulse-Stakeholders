PROJECT OVERVIEW
----------------
HuPulse is a stakeholder engagement and HR analytics platform with a NestJS
backend and a React (Vite) frontend.  The application covers stakeholder
management, communications, surveys/feedback, learning/courses, analytics
dashboards, notifications, tags, and organization settings.

AUTHENTICATION (DEV MODE)
--------------------------
A development bypass chain is in place so the app works without Supabase:
  1. SupabaseGuard   вЂ“ skips JWT validation when NODE_ENV != production
  2. DevAuthMiddleware вЂ“ injects mock user { supabaseUserId: 'dev-user-000' }
  3. OrgResolverMiddleware вЂ“ injects mock context:
       userId = 'dev-user-id'
       organizationId = 'dev-org-id'
       role = 'ADMIN'

DATABASE STATUS
---------------
DATABASE_URL in .env contains placeholder credentials ([YOUR_*]).
PrismaService.onModuleInit() catches connection failures gracefully.
PrismaFallbackInterceptor returns empty payloads (200) for GET requests
when the database is unavailable, so the frontend receives valid responses
instead of 500 errors.
