                         HuPulse Stakeholders – Project Context
================================================================================

PROJECT OVERVIEW
----------------
HuPulse is a stakeholder engagement and HR analytics platform with a NestJS
backend and a React (Vite) frontend. The app is designed for customer success,
program management, and leadership teams to manage stakeholder relationships,
track communications, collect feedback, assign learning, and monitor risk in
one central experience.

USE CASE
--------
This product is built to help enterprise teams maintain stakeholder context,
centralize documentation, coordinate surveys and learning, and expose analytics
insights from a single platform. It is intended as a stakeholder intelligence
hub rather than a basic CRM.

TECHNOLOGY STACK
----------------
Backend:
  • NestJS 10.x  (TypeScript)
  • Prisma 5.x ORM  →  PostgreSQL (Supabase)
  • Passport JWT authentication
  • Swagger / OpenAPI
  • @nestjs/throttler for rate limiting
  • BullMQ + Redis (configured but optional)

Frontend:
  • React 18.3.1
  • Vite 6.3.5
  • TypeScript
  • Tailwind CSS 4.x
  • Radix UI (via shadcn/ui)
  • MUI 7.x (partial use)
  • Recharts (charts)
  • React Router 7.x

AUTHENTICATION (DEV MODE)
--------------------------
A development bypass chain is in place so the app works without Supabase:
  1. SupabaseGuard   – skips JWT validation when NODE_ENV != production
  2. DevAuthMiddleware – injects mock user { supabaseUserId: 'dev-user-000' }
  3. OrgResolverMiddleware – injects mock context:
       userId = 'dev-user-id'
       organizationId = 'dev-org-id'
       role = 'ADMIN'

DATABASE STATUS
---------------
DATABASE_URL in .env contains placeholder credentials ([YOUR_*]).
PrismaService.onModuleInit() catches connection failures gracefully.
PrismaFallbackInterceptor returns empty payloads (200) for GET requests
when the database is unavailable, so the frontend receives valid responses
instead of 500 errors.

DEV SERVER PORTS
----------------
  Backend:   http://localhost:4000   (NestJS)
  Frontend:  http://localhost:5173   (Vite dev server)
  Proxy:     /api/v1 on 5173 → localhost:4000

API SERVICE LAYER  (Frontend/src/app/api.ts)
--------------------------------------------
Exports typed API namespaces:
  • stakeholdersApi   – list, get, create, update, remove, stats, matrix
  • communicationsApi – list, get, create, update, remove, stats
  • surveysApi        – list, get, create, update, remove, submitResponse,
                        results, alerts
  • coursesApi        – list, get, create, update, remove, assign,
                        updateProgress, teamProgress
  • analyticsApi      – dashboard, engagement, risk, departments,
                        communication, predictive
  • notificationsApi  – list, markRead, markAllRead
  • tagsApi           – list, create, remove

All use a shared request<T>(path, init?) helper with BASE = '/api/v1'.

KNOWN LIMITATIONS
-----------------
  • No real database connection – all data is empty/fallback.
  • Frontend pages fall back to hardcoded static data when API returns empty.
  • Redis/BullMQ not configured – notification queues won't function.
  • Supabase auth is bypassed in dev; production auth is untested.
